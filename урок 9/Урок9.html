AI-Ассистент 🤖 WinForms + DeepSeek API

Цель: создать простое приложение на C# с графическим интерфейсом (WinForms), которое отправляет текст на AI-сервис DeepSeek и получает ответ прямо в окне программы.

Теоретическая часть

В этой части мы разберём ключевые понятия 👇

🌐 API (Application Programming Interface) ▾
🔹 API (Application Programming Interface) — это набор правил и соглашений, который позволяет разным программам взаимодействовать между собой. Проще говоря, это «переводчик» между приложениями, который знает, какие команды и данные можно отправлять и получать.
📡 HTTP-запрос ▾
🔹 HTTP-запрос — это способ отправить информацию по сети. По аналогии с заказом в ресторане: мы делаем «заказ» (запрос) и ждём, пока «официант» (сервер) не принесёт «блюдо» (ответ). В случае DeepSeek наш запрос — это запрос через интернет, в котором мы передаём текст, а сервер возвращает ответ в формате данных.
🤖 DeepSeek ▾
🔹 DeepSeek — это современная нейросеть (система искусственного интеллекта), способная обрабатывать естественный язык и генерировать ответы на вопросы. По сути, мы используем DeepSeek как «умного помощника», который на основе большого объёма данных отвечает на наши сообщения.
📄 JSON ▾
🔹 JSON — распространённый текстовый формат для обмена данными. Ответы от API обычно приходят именно в формате JSON. JSON выглядит как список пар ключ-значение: это легко читать и удобно парсить программно.
🔑 API-токен (ключ) ▾
🔹 API-токен (ключ) — это секретный код, который нужен для доступа к DeepSeek. Токен действует как «пароль»: он показывает серверу, что вы — зарегистрированный пользователь с разрешением задавать вопросы. Без токена DeepSeek не позволит получить ответ на запрос.

Таким образом, чтобы использовать DeepSeek, наше приложение должно отправить HTTP-запрос в интернет на специальный адрес DeepSeek API, передав в нём текст вопроса и наш токен. Сервер обработает запрос и вернёт ответ в формате JSON — из него мы извлечём нужный текст и покажем пользователю.

Глава 2 — Написание обработчика кнопки и отправка запроса

1
Дважды щёлкните по кнопке buttonAsk в дизайнере. Visual Studio автоматически создаст метод buttonAsk_Click в коде формы. Это будет наш обработчик события нажатия кнопки.
2
Внутри метода buttonAsk_Click напишем код отправки запроса. Алгоритм такой:
  • 1.получить текст из textBoxQuestion
  • 2.сформировать JSON-запрос
  • 3.послать его на DeepSeek API
  • 4.получить ответ (строку JSON)
  • 5.распарсить JSON и вывести содержимое ответа в labelAnswer
3
Код внутри buttonAsk_Click (C#):
📄 Form1.cs — buttonAsk_Click 
private void buttonAsk_Click(object sender, EventArgs e)
{
    string prompt = textBoxQuestion.Text; // Текст вопроса от пользователя

    string apiKey = "ВАШ_ТОКЕН"; // Замените на свой токен DeepSeek

    // Формируем тело запроса в виде JSON-строки
    string jsonBody =
        "{\"model\":\"deepseek-chat\",\"messages\":[" +
        "{\"role\":\"system\",\"content\":\"Ты помощник для чайников.\"}," +
        "{\"role\":\"user\",\"content\":\"" + prompt + "\"}" +
        "]}";

    // Создаём клиент для отправки HTTP-запроса
    using (WebClient client = new WebClient())
    {
        client.Headers.Add("Authorization", "Bearer " + apiKey);
        client.Headers.Add("Content-Type", "application/json");

        // Отправляем POST-запрос и получаем ответ в формате JSON (строка)
        string response = client.UploadString("https://api.deepseek.com/chat/completions", jsonBody);

        // Разбираем JSON-ответ и извлекаем текст от AI
        using JsonDocument doc = JsonDocument.Parse(response);

        string aiText = doc.RootElement
            .GetProperty("choices")[0]
            .GetProperty("message")
            .GetProperty("content")
            .GetString();

        // Выводим ответ в метку на форме
        labelAnswer.Text = aiText;
    }
}

Разбор кода buttonAsk_Click — по шагам

1
Сначала из поля ввода textBoxQuestion считывается текст, который ввёл пользователь. Это и есть вопрос, который мы отправим нейросети. Текст сохраняется в строковую переменную prompt, чтобы с ним было удобно работать дальше.
2
В переменной apiKey указывается API-токен DeepSeek. Этот токен необходим для доступа к сервису и играет роль пароля. Без него сервер не позволит отправлять запросы. Токен всегда должен храниться в коде аккуратно и не передаваться другим людям.
3
Формируется тело запроса в формате JSON. В JSON мы указываем модель deepseek-chat и массив messages. В массиве сообщений первым идёт сообщение с ролью system — оно задаёт поведение нейросети (мы говорим AI, что он «помощник для чайников»). Второе сообщение имеет роль user и содержит реальный текст вопроса пользователя из переменной prompt.
4
Создаётся объект WebClient — простой класс для отправки HTTP-запросов через интернет. С помощью заголовков передаём серверу два важных параметра: Authorization с нашим токеном доступа и Content-Type, указывающий, что данные отправляются в формате JSON. Без этих заголовков сервер не примет запрос.
5
Метод UploadString отправляет POST-запрос на адрес DeepSeek API и передаёт сформированный JSON. В ответ сервер возвращает строку, которая тоже является JSON-документом. В этом документе содержится сгенерированный ответ нейросети.
6
Чтобы получить сам текст ответа, используется класс JsonDocument. Он позволяет разобрать JSON-строку и обращаться к её элементам. Ответ AI находится внутри массива choices, затем в объекте message, и далее в поле content. Именно это поле содержит текст, который сгенерировала нейросеть.
7
В конце работы метода полученный текст сохраняется в переменную aiText и выводится в элемент labelAnswer на форме. В результате пользователь видит ответ AI прямо в окне программы.

Таким образом, данный метод выполняет полный цикл работы AI-ассистента: получает текст от пользователя, отправляет его в нейросеть через API, принимает ответ и отображает его в интерфейсе Windows-приложения.
