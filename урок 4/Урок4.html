Цель: научиться применять строки, условия и циклы вместе для создания простой алгоритмической задачи — взлом цифрового сейфа. Закрепить счётчики попыток и логику блокировки.

Легенда (вход в урок): Представьте, что вы — хакеры, пытающиеся открыть цифровой сейф. У сейфа есть код из 4 цифр, и у вас ограниченное число попыток, чтобы подобрать правильную комбинацию. Если исчерпаете все попытки, данные в виртуальном сейфе удалятся! В конце урока мы создадим программу-сейф, которая генерирует секретный код и проверяет вводимые коды, как настоящий сейф.

Блок 2 — Алгоритм взлома сейфа

Теперь перейдем к логике «взлома сейфа». Нам предстоит спланировать алгоритм, который будет проверять правильность вводимого кода и ограничивать попытки. Обсудим по шагам, как должна работать наша программа:

1
Генерация кода. Сейф в начале «загадывает» секретный код — случайное 4-значное число. Код хранится внутри программы в строковой переменной, и пользователь его не видит.
2
Ввод и сравнение. Программа запрашивает у пользователя ввод предполагаемого кода. Введённую строку программа сравнивает с секретным кодом.
3
Проверка условия. Если введённый код совпадает с загаданным — выводится сообщение об успешном взломе. Если не совпадает — сообщение об ошибке и уменьшение оставшихся попыток.
4
Повтор или блокировка. Шаг ввода повторяется до тех пор, пока либо пользователь не угадает правильный код, либо не исчерпается лимит попыток (например, 5 попыток). Как только попытки кончаются до того, как код угадан — сейф «самоуничтожается».
5
Состояние «заблокировано». После 5 неправильных вводов наш виртуальный сейф будет делать вид, что удаляет все данные в нём. В реальных устройствах после нескольких неправильных вводов обычно включается защита.

Обратите внимание: мы будем использовать счётчик попыток — специальную переменную, которая увеличивается каждый раз, когда пользователь вводит неправильный код. Это сочетание условий (код не угадан и ещё остались попытки) послужит условием работы цикла. Как только одно из условий нарушится (угадали код или закончились попытки) — цикл проверки прекращается.

Глава 1 — Генерация кода

Начнём с того, что сейф будет генерировать случайный 3-значный код. Используем генератор случайных чисел и сохраним код в строку:

📄 Program.cs — Глава 1 
Random rand = new Random();

string secretCode = "";

for (int i = 0; i < 3; i++)
{
    secretCode += rand.Next(1, 3);
}

Console.WriteLine("Сейф заперт! Загадан секретный код из 3 цифр...");
📷 Картинка 4.1

Объяснение: Сначала мы создаём объект Random для генерации случайных чисел. Затем заводим пустую строку secretCode, куда будем постепенно добавлять цифры кода. С помощью цикла for повторяем 3 раза: rand.Next(1, 3) генерирует случайное число от 1 до 2 (каждая такая цифра станет частью кода), и оператор += добавляет эту цифру в конец строковой переменной secretCode.

После выполнения цикла в secretCode будет храниться, например, "121" — случайная комбинация из 3 цифр. Обратите внимание: сам код мы не показываем пользователю, иначе взломать сейф не составит труда! 😉 Пользователь должен отгадать код самостоятельно.

🔍

Глава 2 — Проверка совпадений

Теперь реализуем ввод кода пользователем и сравнение с секретом. Нам понадобится цикл, который позволит вводить новые попытки до успешного угадывания. Будем читать строку от пользователя и сравнивать её с secretCode:

📄 Program.cs — Глава 2 
string userCode = "";

while (userCode != secretCode)
{
    Console.Write("Введите код: ");
    userCode = Console.ReadLine();

    if (userCode == secretCode)
    {
        Console.WriteLine("Код верный! Сейф открыт.");
    }
    else
    {
        Console.WriteLine("Неверный код, попробуйте еще раз.");
    }
}
📷 Картинка 4.2

Объяснение: Здесь мы объявили переменную userCode для ввода пользователя (изначально пустая строка, чтобы условие цикла while сразу было истинным и запросился ввод). Цикл while (userCode != secretCode) будет повторяться, пока введённый код не совпадёт с секретным. Внутри цикла Console.ReadLine() читает строку, введённую с клавиатуры.

  • Если userCode == secretCode — строка полностью совпала с загаданным кодом: ✅ Код верный! Сейф открыт. — пользователь угадал.
  • Иначе — выполняется блок else: ❌ Неверный код, попробуйте еще раз. — попытка не удалась.

Фактически мы реализовали бесконечный подбор кода до успеха. Но реальный сейф так не работает — у вас не было бы бесконечного количества попыток. Поэтому следующий шаг — добавить ограничение на число попыток.

🛡️

Глава 3 — Защита сейфа (ограничение попыток)

Добавим счётчик и лимит попыток. Пусть у нашего сейфа будет максимум 5 попыток для ввода кода. После 5 неверных вводов сейф блокируется. Внесём изменения в код цикла и добавим переменные для попыток:

📄 Program.cs — Глава 3 
Random rand = new Random();

string userCode = "";
string secretCode = "";
int attempts = 0;
int maxAttempts = 5;

for (int i = 0; i < 3; i++)
{
    secretCode += rand.Next(1, 3);
}

Console.WriteLine("Сейф заперт! Загадан секретный код из 3 цифр...");

while (userCode != secretCode && attempts < maxAttempts)
{
    Console.Write("Введите код: ");
    userCode = Console.ReadLine();
    attempts++; // засчитываем попытку ввода

    if (userCode == secretCode)
    {
        Console.WriteLine("Код верный! Сейф открыт.");
    }
    else if (attempts < maxAttempts)
    {
        Console.WriteLine(" Неверный код. Попытка {0}/{1}.", attempts, maxAttempts);
        Console.WriteLine("-------------------------");
    }
}

if (userCode == secretCode)
{
    Console.WriteLine("Поздравляем! Вы успешно открыли сейф.");
}
else
{
    Console.WriteLine(" Попытки закончились. СЕЙФ ЗАБЛОКИРОВАН! ");
}
📷 Картинка 4.3

Объяснение: Теперь условие цикла стало сложнее: while (userCode != secretCode && attempts < maxAttempts). Цикл будет работать, пока верны оба условия: пользователь ещё не угадал код и число попыток меньше максимума (5).

  • Внутри каждого прохода мы сразу увеличиваем счётчик attempts++ после ввода.
  • В сообщении об ошибке показываем номер попытки через Console.WriteLine("... {0}/{1}", attempts, maxAttempts) — вместо {0} подставится значение attempts, а вместо {1} — maxAttempts.
  • Условие else if (attempts < maxAttempts) — выводим «Неверный код...» только если ещё остались попытки. Это предотвращает вывод лишнего текста после последней попытки.

Когда цикл while завершается — это означает либо код угадан, либо исчерпан лимит попыток. После цикла с помощью заключительного if/else проверяем, чем он закончился: угадали или закончились попытки.

Глава 4 — Самоуничтожение

Теперь добавим финальный эффект, чтобы сейф выглядел по-настоящему «хакерским». В реальных системах после нескольких неправильных попыток доступа часто срабатывает защита: система может заблокироваться, вывести служебные сообщения или показать внутреннюю структуру. Мы сделаем безопасную имитацию такого поведения — после блокировки сейфа программа откроет командную строку Windows и выполнит команду dir /s.

ℹ️

Команда dir /s выводит список всех папок и файлов, начиная с текущей директории, рекурсивно. Она ничего не удаляет и ничего не ломает, а только показывает структуру файлов. Для пользователя это выглядит как серьёзная системная операция.

Чтобы запустить командную строку из C#, нам понадобится класс Process. В начале файла Program.cs добавим строку:

📄 Program.cs — using 
using System.Diagnostics;

Теперь дополним блок, который выполняется, если сейф заблокирован:

📄 Program.cs — Глава 4 
else
{
    Console.WriteLine(" Попытки закончились. СЕЙФ ЗАБЛОКИРОВАН!");

    Process.Start(new ProcessStartInfo
    {
        FileName = "cmd.exe",
        Arguments = "/k cd / && dir /s", // выполняет cd /, затем dir /s
        UseShellExecute = true
    });
}
📷 Картинка 4.5

Объяснение: Здесь мы снова используем условие if (userCode != secretCode). Оно означает: если код так и не был угадан, значит, пользователь израсходовал все попытки.

  • Process — это класс, который позволяет запускать внешние программы из C#.
  • FileName = "cmd.exe" — указываем, что хотим запустить командную строку Windows.
  • Arguments = "/c dir /s" — говорим командной строке: /c — выполнить команду и закрыться, dir /s — вывести список всех файлов и папок.
  • UseShellExecute = true — разрешаем Windows открыть командную строку стандартным способом (в отдельном окне).

В результате, сразу после сообщения о блокировке: открывается окно командной строки, в нём автоматически выполняется команда dir /s, пользователь видит поток системного вывода — как будто система начала внутреннюю проверку. Важно: никакие файлы не удаляются, это только визуальный и обучающий эффект.

⚠️

МОЖНО ДЕТЯМ ПОКАЗАТЬ В КОНЦЕ команду shutdown /s /t 1. Её нужно написать вместо dir /s — и у вас отключится компьютер через секунду 😈
